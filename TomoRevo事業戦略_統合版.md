## 1. 存在意義（Why）

### Mission（存在意義）
全てのステークホルダーのワクワクの総量を最大化することに貢献する。
社長・社員の自己実現の場を価値として提供する。

### Vision（目指す未来）
**子供たちが大人に憧れて育つ社会の実現。**

ひとりひとりの大人が人生の舵を自分で握り、日々ワクワクの感情に満ちた人生を過ごしている状態。
「仕事＝自己実現」という価値観が当たり前になった世界。

### Value（提供価値）
テクノロジーという手段を通じて **「個人の想いを最大化する」**。
単純作業という鎖から人々を解き放ち、大人が「ワクワク」を原動力に生きる姿を見せることで、次世代に希望を与える社会を創る。

## 2. 個人の理念（代表・藤田裕二）

### 人生理念：愛・感謝・ワクワク

**愛**: 大切な人への想いやりの心であり、人の内発的動機の源となる最も大きなエネルギー。自分自身を愛し、他者にも同じように愛を持った関わりを選択する。

**感謝**: 事実に対して自分自身の成長を選択すること。身の回りに起こる全ての事象に対して感謝の気持ちを持って選択する。

**ワクワク**: これからの未来に対して想像を膨らませ、希望を抱き楽しみにしている状態。日々肯定的選択をし、自分自身のビジョンや価値観に沿った選択をし続ける。

### 人生ビジョン
子供たちが大人に憧れて育つ社会の実現。ワクワクを生きる大人を自分自身で体現し続ける。

### 愛の定義
> 愛とは他者への思いやりの心であり、人の内発的動機づけに最も大きなエネルギーを与えるもの

### 理念の構造（順番がある）
```
愛を受け取れる器になる → 感謝が生まれる → ワクワクできる
（自己愛）            （過去の再解釈）    （未来への希望）
```

## 3. 解決すべき課題

### 構造的問題
熱い思いと確かなサービス力を持つ事業者が、本来集中すべき「価値創造」ではなく「業務処理」に時間を奪われ続けている。これは個人の能力や意識の問題ではなく、事業運営の構造的な問題。

### 負の連鎖
```
忙しい → 改善できない → スケールしない → 疲弊する → また忙しい
```

### さらに深い課題（選択理論の視点）
表面的には「忙しい」だが、本質的には：
- 自分が本当に大切にしたいことが見えなくなっている
- 「耐えの期間」と割り切って、大切な人や自分を蔑ろにしている
- ネガティブ感情が負の行動に変換されている（自己嫌悪ループ、思考停止）
- 致命的な7つの習慣を他人にはしないが、自分にはやり続けている

**解くべきは「人」ではなく「仕組み」。しかし、仕組みだけでは本質は変わらない。「想い」と「仕組み」の両輪が必要。**

## 4. ターゲット

### メインターゲット
**ビジョン駆動 × IT苦手 × 少人数事業のオーナー**

- 1人社長〜少人数チーム（概ね10名以下）
- 熱い想いで事業を始めたが、業務処理に追われている
- AIへの関心はあるが独力では使いこなせない
- 美容・ウェルネス・コーチング・教育・オンラインサービス系が中心になりやすい
- 結果として女性が多い（が、性別では絞らない）

### なぜこのターゲットか
1. **実績がある** — 初クライアント（40後半女性、半年40万円コーチング）、現クライアントも同層
2. **全武器が活きる** — コーチング×SE×AI×選択理論の掛け合わせが100%刺さる
3. **競合がいない** — IT苦手な情熱的事業者向けの「寄り添い型」AI伴走サービスは空白地帯
4. **ビジョンに直結** — ワクワクと生きる大人を直接増やせる

### ターゲットのペルソナ
熱い思いを持って事業を始めたのに、気づけば事務作業や目の前の生産に追われる日々。大切なものや人を蔑ろにしてしまっていることに気づいてはいるが、「耐えの期間」と割り切っている。でも心が削られていて、たまに限界を迎える。

> あなたが本当に求めていたものは今ここにありますか？
> もっと準備ができてから。そう言って見送ってきたチャンス、いくつありましたか？
> これからの人生において準備が整う時期なんてやってこない。

### 検証方針
既存クライアント2名への提案結果をもとにターゲット・価格を調整する。答えは市場にある。

## 5. コンセプト

### サービス名称
**ITコーチング**

クライアントの言葉から生まれた名称：
- 「IT全般をお願いしてる」（Bさん）+ 「話してるだけでコーチングみたいになってる」（めぐさん）= ITコーチング
- 一言で何屋かが伝わる。説明不要。

### 何をする人か（一言）
**あなたの"こうしたい"を引き出して、そのままITで形にする人。**

想いをITに翻訳する。他のWeb制作屋は「仕様書ください」から始まる。ユウジさんは「何がしたい？」から始まる。

### コアメッセージ
**入口は機能訴求。本質的価値は体験の中で伝わる。**

- 入口（表面的ニーズ）: LINE構築、予約システム、LP制作 → わかりやすい機能的価値
- 体験（想定を超える）: 「こんなに寄り添ってもらえると思わなかった」
- 継続（離れない理由）: 「人柄」「寄り添い」→ ユウジさんにしかできない情緒的価値
- 拡大（信頼からの展開）: 「実は他にもこういうの困ってて...」

コピーは市場検証後に確定する。まず提案し、クライアントがどんな言葉に目を輝かせるかを見る。

### ポジショニング
- AI導入支援ではなく、**AI組織構築**
- ツール導入ではなく、**仕組み設計**
- システム販売ではなく、**想いの伝達を生み出す**
- クライアントに学習コストを一切負わせない（内部方針：「ITを使わないDX屋」）

### 核心原則
TomoRevoは単に技術を売るのではなく、クライアントの **「想いの伝達」** を生み出すことを真の提供価値とする。テクノロジー導入の摩擦を排除し、クライアントが自身の **「天手業（本業）」** に100%集中できる環境を構築する。

### 真の差別化（クライアントの声から）
> 「IT業界のベテランさんとかも他に知ってるけど、ゆうじくんに頼んでるのは人柄よ」

**機能は真似できる。寄り添いは真似できない。** LINE構築は誰でもできるが、「こんなに寄り添ってもらえると思わなかった」はユウジさんの「引き出す力」と「愛ある関わり」からしか生まれない。

## 6. サービス設計

### 全体構造

```
ユウジさん:  セッション（引き出す・方向づけ・意思決定）= 0→1, 70→100
チーム:     DX・AI実装、仕組み構築、運用サポート     = 1→70
AI:        作業自動化                             = 1→70の加速装置
```

### サービスの流れ

#### 初回: ヒアリング＆即日制作（入口）
- 「LINE構築したい」「予約システム作りたい」等の機能的ニーズで入ってくる
- ヒアリングの中で自然と「本当に大切にしたいこと」が引き出される（＝選択理論ベースの対話。表には出さない）
- その場でたたきを作って「え、もうできたの？」という体験を提供
- **この初回体験で「寄り添い」の価値が伝わり、継続につながる**

#### Step 1: 事業オペレーション設計
- 業務フロー・予約・顧客管理導線の整理
- 情報構造の整理と可視化
- ツールの最適な組み合わせ設計

#### Step 2: 自動化・AI構築
- 予約・顧客対応の自動化
- リマインド・管理業務の効率化
- 各社オーダーメイドのAIエージェント環境構築
- 企業専用AI（社長AI・営業AI・マーケAI・ナレッジAI等）

#### Step 3: AI文化の定着・伴走
- チーム全体へのAI活用浸透
- AIを「ツール」ではなく「思考パートナー」として使う文化づくり
- 継続的な改善・運用サポート
- 定期セッションでの経営者伴走

### WEB制作（即効性のある入口サービス）
- **キャッチコピー: 「その日にできるWEB制作」**
- LP / HP / 公式LINE / Instagram
- ヒアリング→たたき作成→実装を1日で完結
- AIチームが実作業を担当

### 4つの柱（サービスカテゴリ）

| 柱 | 内容 | 提供形態 |
|----|------|---------|
| 事業オペレーション設計 | 業務フロー・情報構造・ツール設計 | 初期プロジェクト型 |
| 自動化・AI構築 | 予約・対応・レポートの自動化 | 初期プロジェクト型 |
| AIオペレーションチーム伴走 | 継続運用・改善・チーム浸透支援 | 月額サブスク（継続） |
| AI活用文化定着支援 | チームへのAI活用研修・定着 | ワークショップ・伴走型 |

### 将来事業: CTO派遣
- TomoRevoからCTO代行を派遣
- 専門領域×最強のAIチーム（例：デザイナー×AI、コーチング×AI）
- 実作業はAIチームが担当

## 8. 差別化

### 他社との違い

| 一般的なDXコンサル | TomoRevo |
|------------------|----------|
| ツール導入（点） | 仕組み設計（面） |
| 「業務を効率化します」 | 「あなたが大切にしたい時間を取り戻します」 |
| IT知識が前提 | 学習コスト一切なし |
| コンサルだけ or 実装だけ | コンサル×実装×伴走の一体型 |
| 機能的価値 | 選択理論ベースの本質的変容 + 機能的価値 |

### なぜTomoRevoにしかできないか
1. **IT/SEとしてのシステム目線**がある
2. **対人サービスの現場目線**（人の気持ち・温かみ・理想と実務のギャップ理解）
3. **AI活用の実践者**（自分の課題をAIで一つずつ解消している実績）
4. **コーチングスキル**（天性＋後天的学習）
5. **選択理論心理学**を本気で学んでいる
6. → 全て掛け合わせ＝唯一無二のサービス

## 12. ビジョンと社会への想い

事業者の解放 → 価値の連鎖 → 社会への波及
