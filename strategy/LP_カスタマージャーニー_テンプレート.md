## 1. 認知フェーズ — LPとの出会い

### 流入経路と心理状態

| 流入元 | 想定される状況 | 心理状態 | 温度感 |
|--------|-------------|---------|--------|
| **Instagram（リール・ストーリーズ）** | 「1人事業者あるある」「事務作業に追われて本業できない」系の投稿に共感 → プロフィールリンクへ | 「わかる...」と共感しているが、まだ他人事。情報収集モード | 低〜中 |
| **紹介（既存クライアント・知人）** | 「こういうの困ってない？いい人いるよ」と紹介された | 信頼する人からの推薦で安心感あり。ただし「とりあえず見てみるか」 | 中〜高 |
| **Google検索** | 「個人事業 IT 何から」「LP 作りたい わからない」「DX 一人事業者」等 | 課題が顕在化している。解決策を探している。比較検討モード | 中 |
| **YouTube動画からの導線** | ユウジさんの発信を見て「この人に相談したい」と思った | 人柄への信頼が既にある。サービス内容を確認したい | 高 |

### 流入経路別の対策

- **Instagram経由**: LPのファーストビューで「あ、あの投稿の人だ」とつながる一貫性が必要
- **紹介経由**: 紹介者の信頼を引き継ぐ設計（「〇〇さんからのご紹介ですか？」の導線があると理想）
- **検索経由**: ファーストビューで「何をしてくれるサービスか」が3秒で伝わること
- **YouTube経由**: 人柄は伝わっている前提。サービスの具体性と料金に到達しやすい動線

### セクション1: ファーストビュー（滞在目安 3〜5秒）

**目的**: 「自分のためのページだ」と感じさせ、スクロールさせる

**読み手の心理**:
- 「何のサービス？」
- 「自分に関係ある？」
- 「怪しくない？」

**必要な要素**:
- キャッチコピー: 「あなたの『こうしたい』を引き出して、そのままITで形にする。」
- サブコピー: 誰のための何のサービスかを1行で
- ユウジさんの写真（信頼感・人柄が伝わるもの）
- CTAボタン（まずは無料で相談する）

**判定基準**: 3秒以内に「もう少し読んでみよう」と思えるか

---

### セクション2: 悩みの共感（到達率目安 70〜80%）

**目的**: 「自分のことだ」と感じさせる

**読み手の心理**:
- 「あ、これ完全に自分のことだ...」
- 「やっぱりみんなそうなんだ」
- 「わかってくれる人がいるんだ」

**現在の提案資料の該当部分**:
> - 熱い想いで事業を始めたのに、気づけば事務作業に追われている
> - 本当にやりたいことに集中する時間がない
> - 「もっと準備ができてから」と思って、気づけば1年経っている

**LP化のポイント**:
- 5つの悩みは多すぎない。3つに絞って刺さりを深くする選択肢もあり
- 「それは、あなたの能力や情熱の問題ではありません。」 ← この一文が心理的安全性を作る。LP上で視覚的に強調する

**感情変化**: 不安・モヤモヤ → 「わかってもらえている」安心感

---

### セクション3: 解決策の提示 — ITコーチングとは（到達率目安 50〜60%）

**目的**: 「こういう方法があったのか」と新しい選択肢を認識させる

**読み手の心理**:
- 「普通のIT業者とは違うの？」
- 「コーチングとITって何の関係？」
- 「ITを覚えなくていいってどういうこと？」

**現在の提案資料の強い一文**:
> 普通のIT業者は「仕様書ください」から始まります。TomoRevoは「何がしたいですか？」から始まります。

**LP化のポイント**:
- 比較表は非常に効果的。LP上ではビジュアルで差別化を際立たせる
- 「あなたはITを覚える必要はありません。本業に集中してください。」 ← これがターゲットの最大の心理障壁を外す一文。大きく配置

**感情変化**: 疑問 → 「自分にもできるかもしれない」という可能性の認識

---

### セクション4: なぜTomoRevoにできるのか（到達率目安 40〜50%）

**目的**: 信頼の根拠を示す

**読み手の心理**:
- 「本当にできるの？」
- 「どんな人がやってるの？」
- 「実績は？」

**現在の提案資料の構成**:
- 経歴テーブル（ジム運営 × SE × YouTube × マーケ × CTO × コーチング）
- お客様の声3件

**LP化のポイント**:
- 経歴は「だからあなたの気持ちがわかる」に接続させる。スキルの羅列ではなく物語として見せる
- お客様の声は最強の信頼構築。可能なら写真付き・実名（許可取得前提）
- 「IT業界のベテランさんも他に知ってるけど、ゆうじくんに頼んでるのは人柄よ」 ← この声が人柄訴求と一致する

**感情変化**: 懐疑 → 「この人なら信頼できそう」

---

### セクション5: 実績・Before/After（到達率目安 35〜45%）

**目的**: 具体的な変化をイメージさせる

**読み手の心理**:
- 「実際にどう変わるの？」
- 「自分もこうなれる？」

**LP化のポイント**:
- Before/Afterは最も説得力のあるコンテンツ。できれば2〜3件に増やす
- 数字で語れる部分は数字で（「月〇時間の業務削減」「新規集客〇%増」等）
- 定性的な変化も大切（「大切なことに集中できる時間を取り戻した」）

**感情変化**: 「なるほど」 → 「自分もこうなりたい」という欲求の芽生え

---

### セクション6: サービス内容・料金（到達率目安 25〜35%）

**目的**: 「何をしてくれるか」「いくらか」の透明性

**読み手の心理**:
- 「具体的に何をしてくれるの？」
- 「いくらかかるの？」（最大の関心事であり最大の不安）
- 「自分にはどのプランが合うの？」

**LP化のポイント**:
- 料金を出すかどうかは戦略的判断が必要
  - **出すメリット**: 透明性で信頼獲得。予算が合わない人の問い合わせを事前フィルタリング
  - **出さないメリット**: 料金だけで判断されない。面談で個別提案できる
  - **推奨**: 料金レンジ（月額5万円〜）は提示し、「あなたに合ったプランを一緒に決めましょう」で面談に誘導
- 「対話 → IT実装 → 伴走」の3ステップ図はそのまま使える。LP上ではアイコン+短文で視覚化

**感情変化**: 期待 → 現実的な検討モードへ移行（ここで離脱or次のステップへ分岐）

---

### セクション7: 代表メッセージ（到達率目安 20〜30%）

**目的**: 最後の一押し。人柄と共感で決断を後押しする

**読み手の心理**:
- 「この人に自分のことを任せたい」
- 「同じ経験をした人だからわかってくれそう」

**現在の提案資料の強い部分**:
> 私自身、1人でジムを運営していた時、事務作業に追われて本当にやりたいことに時間が使えない日々を過ごしていました。

**LP化のポイント**:
- ここはユウジさんの「飾らないのに芯がある」人柄がそのまま出る場所。過剰な演出は不要
- 写真は笑顔で、柔らかい雰囲気のもの

**感情変化**: 信頼 → 「この人と話してみたい」

---

### セクション8: CTA — お問い合わせ（最終到達）

**目的**: 行動に移させる

**読み手の心理**:
- 「相談してみようかな...でも何を話せばいいかわからない」
- 「まだ自分には早いかも」
- 「無料なら...」

**LP化のポイント**:
- 「何を相談すればいいかわからない、でも大丈夫です」← この一文は必須。最大の心理障壁を外す
- CTAボタンのコピー例: 「まずは無料で話してみる」（押しが強すぎない表現）
- フォームは最小限（名前、連絡先、簡単な状況の3項目程度）
- LINEで友達追加 → そのままチャットで相談開始、の導線も検討

## 4. CTA後の導線 — 問い合わせから契約まで

```
問い合わせ                初回面談              提案                  契約
  │                        │                   │                    │
  ▼                        ▼                   ▼                    ▼
フォーム送信             30〜60分            個別プラン提案         契約・決済
or LINE追加            オンライン対話         （面談後1週間以内）    サービス開始
  │                        │                   │                    │
  ▼                        ▼                   ▼                    ▼
自動返信で安心感       「この人に任せたい」    「自分に合っている」    「やっと動き出せる」
日程調整リンク送付      と確信する時間         と納得する時間          というホッとした感覚
```

### Step 1: 問い合わせ受付（即日〜24時間以内）

**ユーザーの感情**: 「送ったけど大丈夫かな...」（不安）

**やること**:
- 自動返信メッセージ: 「ありがとうございます。〇営業日以内にご連絡します」
- 24時間以内にユウジさんから個別メッセージ
- 日程調整ツール（Calendly等）で初回面談の日程を確定

**ポイント**: レスポンスの速さ = 信頼の第一歩。自動化で確実に対応

### Step 2: 初回面談（無料・30〜60分・オンライン）

**ユーザーの感情**: 「何を話せばいいんだろう」（緊張） → 「こんなに聞いてくれるんだ」（安心・感動）

**やること**:
- 売り込みゼロ。完全にヒアリングに徹する
- 「何がしたいですか？」「何に困っていますか？」から始める
- 現状の課題を整理し、可能性を提示する
- 最後に「こういう形でサポートできますがどうですか？」と提案への接続

**ポイント**: この面談自体が「ITコーチング」の体験版になる。ユウジさんの「引き出す力」が最大の武器

### Step 3: 提案（面談後3日〜1週間以内）

**ユーザーの感情**: 「具体的にどうなるんだろう」（期待と不安の混在）

**やること**:
- 面談内容を踏まえた個別提案書（課題 → 解決策 → スケジュール → 料金）
- どのプランが最適かを理由付きで提案
- 「まずはライトから始めて、合えばスタンダードに」のようなステップ提案も有効

**ポイント**: 汎用テンプレートではなく「あなたの話を聞いて、あなたのために作った提案」であることが伝わること

### Step 4: 契約・開始

**ユーザーの感情**: 「やっと動き出せる」（安堵・ワクワク）

**やること**:
- 契約書の締結（電子契約）
- 決済システムでの支払い開始
- キックオフセッション（初回セッション）の日程確定
- 「Welcome to TomoRevo.」の歓迎メッセージ

**ポイント**: 契約後の最初の体験がその後の継続率を決める。初回セッションまでの間に「始まった」という実感を持たせる

## 6. ITコーチングLP固有の戦略的ポイント

### ターゲットの特徴を踏まえた設計

ターゲットは「熱い思いで事業をしているが、キャパパンパンで緊急タスクに追われている事業者」。

- **時間がない**: LPは長すぎない。核心をコンパクトに伝える（スクロール3〜4回で全体が見える）
- **IT苦手意識がある**: LP自体のデザインがシンプルで見やすいこと = サービスの信頼性を体現
- **押し売りに敏感**: 煽りコピーや緊急性の演出は逆効果。ユウジさんの「飾らないのに芯がある」トーンで
- **人柄で選ぶ**: 写真・動画・お客様の声で「この人と話したい」を作る

### コンバージョン目標の設定

| 指標 | 目標値 | 測定方法 |
|------|--------|---------|
| LP到達 → CTA到達率 | 20〜30% | スクロール率計測 |
| CTA到達 → 問い合わせ率 | 10〜20% | フォーム送信数 / CTA到達数 |
| 問い合わせ → 面談実施率 | 70〜80% | 日程確定数 / 問い合わせ数 |
| 面談 → 契約率 | 50〜70% | パートナーの確信「体験セッションさえ取れば絶対に契約に持ってくる」 |
| **全体CVR（LP到達 → 問い合わせ）** | **2〜5%** | 月間目標: 100PVで2〜5件の問い合わせ |

# Part 2: 汎用LP制作テンプレート & フレームワーク

---

## A. LP構成テンプレート（セクション順と役割）

以下の構成はどんなサービスLPにも適用可能な基本型。必要に応じてセクションの追加・削除・順序変更を行う。

### 基本構成（8セクション）

| # | セクション名 | 役割 | 読み手の問い | 必須度 |
|---|------------|------|------------|--------|
| 1 | **ファーストビュー** | 3秒で「自分ごと」にする | 「何のサービス？自分に関係ある？」 | 必須 |
| 2 | **悩み・課題の共感** | 「わかってくれる」と感じさせる | 「自分のことだ...」 | 必須 |
| 3 | **原因の提示** | 悩みの本質を言語化する | 「だからうまくいかなかったのか」 | 推奨 |
| 4 | **解決策の提示** | サービスの概要を伝える | 「こういう方法があるのか」 | 必須 |
| 5 | **他との違い** | 選ばれる理由を明確にする | 「なぜここを選ぶべき？」 | 推奨 |
| 6 | **信頼の根拠** | 実績・経歴・お客様の声 | 「本当にできるの？」 | 必須 |
| 7 | **サービス詳細・料金** | 具体的な内容と投資額 | 「何をいくらでしてくれる？」 | 必須 |
| 8 | **CTA（行動喚起）** | 次のアクションに導く | 「どうすればいい？」 | 必須 |

### セクション間のブリッジ（接続の原則）

各セクションの終わりに、次のセクションへ読み進める理由を作る。

- 悩み共感 → 「でも、それはあなたのせいじゃない」 → 原因の提示へ
- 原因の提示 → 「だから、こういうアプローチが必要」 → 解決策へ
- 解決策 → 「なぜ私たちにできるのか」 → 信頼の根拠へ
- 信頼の根拠 → 「具体的にはこうサポートします」 → サービス詳細へ
- サービス詳細 → 「まずはここから始めませんか」 → CTAへ

### ファーストビューのフォーマット

```
[キャッチコピー] — 誰の、何の悩みを、どう解決するか（1行）
[サブコピー] — 補足説明（1〜2行）
[ビジュアル] — サービス提供者の写真 or サービスイメージ
[CTAボタン] — 最も低いハードルのアクション（「無料で相談する」等）
```

### CTAセクションのフォーマット

```
[背中を押す一言] — 不安を解消するメッセージ
[CTAボタン] — 明確なアクション（「まずは無料で話してみる」等）
[補足] — ハードルを下げる情報（「〇分で完了」「しつこい勧誘はしません」等）
[フォーム or LINE] — 最小限の入力項目
```

| **行動** | （問い合わせする） | | | | | |
| **体験** | （初回面談を受ける） | | | | | |
| **決断** | （契約する） | | | | | |
| **利用開始** | （サービスを受ける） | | | | | |

### 感情曲線の設計ポイント

1. **谷を作りすぎない**: 感情が下がるポイント（料金表示等）は1箇所に留める
2. **谷の直前に山を作る**: 料金を見せる前に、十分な期待感と信頼を構築する
3. **最後は必ず上向きで終わる**: CTA周辺は「やってみたい」のポジティブ感情で
4. **不安の解消は先回りする**: 読み手が不安を感じる前にその答えを提示する

## D. LP改善の優先順位フレームワーク

LP公開後に改善する際の優先順位。上から順に効果が大きい。

| 優先度 | 改善箇所 | 理由 |
|--------|---------|------|
| 1 | **ファーストビューのコピー** | 最も多くの人が見る場所。ここの改善が全体CVRに最も影響する |
| 2 | **CTAのコピーと配置** | 最終的な行動を左右する |
| 3 | **流入元の質** | LP自体が良くても、関心のない人が来ていたら意味がない |
| 4 | **お客様の声・実績** | 信頼構築の核。数と質を継続的に充実させる |
| 5 | **料金の見せ方** | 表現の仕方で印象が大きく変わる |
| 6 | **ページ表示速度** | 3秒以上かかると離脱率が大幅に上がる |
| 7 | **フォームの項目数** | 項目を1つ減らすだけでCVRが上がることがある |
